# 画像入力機能マニュアル（かんたん版）

動画素材づくりで、**自分で撮った写真を使えるようになりました**。
これまで全部AIで画像を生成していましたが、今後は「実写を優先、足りない分だけAI生成」ができます。

---

## 1. できること（3つ）

| # | できること | ざっくり言うと |
|---|---|---|
| ① | 手持ち写真を素材に使う | 撮った写真をそのままシーンに使える |
| ② | 参考写真に寄せてAI生成 | 「この雰囲気で」とお手本を渡してAIに描かせる |
| ③ | AIが写真を見て判断 | ラベル読み取り・シーン割当・キャプション・変な画像のチェックを自動で |

---

## 2. 使い方（基本はこれだけ）

### ステップ1：写真を置く
その週のフォルダの中に **`my_photos`** という名前のフォルダを作り、写真を入れます。

```
output/2026-07-14-wine/my_photos/
    barolo.jpg
    piedmont_vineyard.jpg
    glass.jpg
```

- フォルダ名は必ず **`my_photos`**（半角）。
- 写真の形式：`.jpg` / `.png` / `.jpeg` / `.webp` / `.gif`。
- ファイル名は日本語でもOKですが、半角英数がおすすめ。
- **枚数は自由**。全シーン分なくてもOK。

### ステップ2：いつもどおり動画素材生成を実行
火曜の動画素材生成（自動）で、動画エージェントが自動的に：
1. `my_photos` の中身を見て、
2. 台本の各シーンに合う写真を配置し、
3. 写真がないシーンだけAIで生成し、
4. 各画像の品質をチェックします。

### ステップ3：結果を確認
できあがった画像は、いつもと同じ場所に入ります。

```
output/2026-07-14-wine/images/
    scene_01.png  ← 実写（barolo.jpg を配置）
    scene_02.png  ← AI生成
    scene_03.png  ← 実写
    ...
```

写真は自動で **1536×1024（16:9）** に整えられるので、AE（After Effects）の取り込みはこれまでと全く同じです。

---

## 3. よくある使い方のコツ

- **写真を置かなければ、これまでどおり全部AI生成**されます。今までの運用を変えずに、使いたい週だけ写真を入れればOK。
- **見た目を統一したいとき**：実写と同じ雰囲気でAI生成させたい場合、代表的な手持ち写真を「お手本（reference）」として渡せます（②）。エージェントに「barolo.jpg の雰囲気に合わせて」と伝えればOK。
- **実物のボトルを登場させたいとき**：手持ちのボトル写真を渡すと、そのボトルを使ったシーンをAIが生成します（②）。
- **ラベルを読ませたいとき**：ワインのラベル写真から銘柄・品種を読み取り、キャプションに反映できます（③）。

---

## 4. Claude Code から手動で使う場合

`@video` エージェントに台本と出力先を渡すと、上記を自動でやってくれます。
個別に頼みたいときは、次のようにツール名で指示できます：

| 頼みたいこと | ツール |
|---|---|
| 写真を全部見て内容を教えて | `scan_photos` |
| この写真をシーンNに使って | `assign_photo` |
| この写真を参考にシーン画像を作って | `generate_scene_image`（reference_image 指定） |
| この画像に何が写ってる？／変じゃない？ | `analyze_image` |

例：「`output/2026-07-14-wine` の my_photos を確認して、合うシーンに配置して。足りない分は生成して」

---

## 5. 注意点

- `my_photos` フォルダは **その週のフォルダの中**（`output/日付-wine/` や `output/日付-coffee/`）に作ります。
- 縦長・横長どちらの写真でも、自動で中央を切り取って16:9に整えます（見切れる場合があるので、大事な被写体は中央寄りだと安心）。
- AI生成・画像解析には従来どおり API キー（`OPENAI_API_KEY` / `ANTHROPIC_API_KEY`）が必要です。未設定でもエラーで止まらず、その処理だけスキップされます。
